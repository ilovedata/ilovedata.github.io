1. Julia 와 R의 범위규칙¶

Julia언어의 범위규칙(scope rule)은 R과 유사하게 정적 범위규칙(lexical scope rule, statisc scope rule)을 따른다. 정적범위규칙은 동적범위규칙(dynamic scope rule)과 다르게 프로그램에 짜여진 구성만으로 변수들의 범위규칙을 정할 수 있다. 범위규칙은 정의된 변수가 어떤 범위에서 어떤 값을 가지고 그 값이 어떻게 변화하는지에 대한 규칙이다.

lexical scope means that the scope of variables can be inferred from the source code alone.(julia manula)

일반적으로 프로그램 언어에서는 변수의 범위 규칙의 경계를 정할 수 있는 일련의 코드로 구성된 블럭이 있으며 이를 범위블럭(scope block)이 있다. 범위블럭은 모듈, 함수, 조건문 등이 될 수 있다. Julia는 두 개의 범위, 전역범위(global scope)와 국소범위(local scope)가 있다. 다음과 같은 두 가지 다른 종류의 범위블럭이 있다. 범위블럭을 생성하는 것들은 다음과 같이 두 종류로 나뉜다.

  • Scope blocks that may nest only in other global scope blocks:
    • global scope:
      • module, baremodule
      • at interactive prompt (REPL)
    • local scope (don't allow nesting):
      • (mutable) struct, macro
  • Scope blocks which may nest anywhere (in global or local scope):
    • local scope:
      • for, while, try-catch-finally, let
      • functions (either syntax, anonymous & do-blocks)
      • comprehensions, broadcast-fusing

1.1 전역범위 (Global scope)¶

가장 기본적이고 큰 범위블럭은 module이며 하나의 module은 다음의 전역범위를 만든다. 즉 module은 별개로 구성된 변수들의 공간이다. 보통 R에서 패키지의 namespace와 유사한 개념이다.

하나의 module에서 정의된 전역범위는 다른 module에 의해 정의된 전역블럭과는 분리된다. 아래에서 module A 에서 정의된 변수 a는 다른 module B 에서 단순하게 접근할 수 없다.

1.2 국소범위(Local scope)¶

국소범위는 대부분의 블럭 구조에 의하여 생성된다. 단 begin과 if는 예외적인 경우이다.

이제부터는 모든 작업이 새로운 module인 REPL안에서 시작한다.

아래 프로그램은 두 변수가 같은 이름 x를 가지만 함수내(블럭내)의 변수는 국소범위를 가진다.

In [4]:
x=1

function bar()
  x=2  # it is local variable
  println(x)
end;

bar()
x

2
Out[4]:
1

명령어 local과 global은 명시적으로 범위를 지정한다.

In [5]:
x=1
 
function foo()
  global x
  x=2
end 

foo()
x
Out[5]:
2

1.3 내포된 함수 (nested function)의 범위규칙¶

Julia에서는 여러 개의 함수가 내포된 경우 상위함수의 범위에 있는 local 변수의 값이 같이 변화한다.

In [6]:
x=1
y=2

function f()
  x=2          # new local x 
  function g()
    x=11       # x here refers to parent local x and its value is changed 
               # "parent x" is also changed to 11
    return x+y # y = 2 is global and return 11+2 (x+y)
  end
  return g()+x # g()= 13 and x= 11 so return 24
end

f()
Out[6]:
24

위에서 나타난 내포된 함수에서 local 변수의 변화는 R과 매우 다른 방식의 범위규칙이다. 동일한 함수를 R 프로그램으로 실행해 보자.

In [7]:
using RCall

R"""
x <- 1
y <- 2

f <- function() {
  x <- 2              # new local x
  g <- function() {
    x <- 11           # another new local x is defined  
                      #so new x=11 but parent x is still 2
    return(x+y)       # y is global and return 11+2
  }
  return(g()+x)       # g()= 13 and x= 2 so return 15
}

f()
"""
Out[7]:
RCall.RObject{RCall.RealSxp}
[1] 15

위에서 본 것과 같이 함수내에서 다른 함수가 정의되었을 때 범위규칙은 함수의 결과값이 함수인 경우 함수의 closure 생성에 매우 큰 영향을 미치게 된다.

Jilia에서는 부모함수(parent)내서 local 변수가 정의되고 자식함수(child)안에서 상위함수와 같은 이름의 local 변수가 나타나면 이를 새로운 local 변수로 보지않고 상위함수에서 정의된 변수로 간주한다(마치 부모함수안에서의 global 변수처럼) 따라서 자식함수내에서 그 변수의 값이 변하면 부모함수의 local 변수가 같이 변화한다.

그러나 R에서는 자식함수내에서 상위함수에서 정의된 같은 이름의 변수가 나타나더라도 이를 새로운 local 변수로 설정한다. 위의 R프로그램에서 함수 g() 에서 나타나 x <- 11에서 x 는 새로운 local 변수로 정의되는 것이다. 따라서 상위함수 f()에서 정의된 local 변수 x는 변하지 않는다. 이렇게 R에서 내포함수의 범위규칙은 일관성이 있다.

Julia에서 위에서 나타난 R의 결과처럼 자식함수에서 정의된 변수 x를 새로운 local변수로 만들 수는 없을까? 이러한 코딩은 가능하며 아래의 포로그램에서 처럼 local이라는 명령어를 변수 선언에 사용하면 된다.

In [8]:
x=1
y=2

function f()
  x=2          # new local x 
  function g()
    local x    # now new local x
    x=11       # new local x is defined 
    return x+y # y is global and return 11+2
  end
  return g()+x # g()= 13 and x= 2 so return 15
end

f()
Out[8]:
15

이제 질문을 바꾸어 R에서 Julia의 내포함수의 범위규칙처럼 부모함수와 자식함수의 local 변수가 같이 지정되는 방법이 있을까??

자식함수에서 변수 x 를 정의할 때 x <<- 11 을 사용하면 이를 구현할 수 있다. 지정자 <<-를 superassignment operator 라고 부르며 함수의 environment(closure)에 있는 변수의 값을 바꿀 수 있다.

In [9]:
R"""
x <- 1
y <- 2

f <- function() {
  x <- 2              # new local x
  g <- function() {
    x <<- 11          # now x refers to parent x  
                      # so  parent x becomes 11
    return(x+y)       # y is global and return 11+2
  }
  return(g()+x)       # g()= 13 and x= 11 so return 24
}

f()
"""
Out[9]:
RCall.RObject{RCall.RealSxp}
[1] 24

2. 결과를 함수로 주는 함수: function as return value¶

혼합모형(Mixed models)에서 같은 임의효과(random effect)를 공유하면 두 개의 확률변수는 다음과 같은 분산과 공분산을 가진다.

$$ y_{ij} = \mu + B_i + e_{ij}, \quad y_{ik} = \mu + B_i + e_{ik} $$ $$ A_i \sim N(0, \sigma_b^2), \quad e_{ij}, e_{ik} \sim_{ind} N(0, \sigma_e^2$$ $$ V(y_{ij}) = V(y_{ik}) = \sigma^2_b + \sigma^2_e, \quad Cov(y_{ij}, y_{ik} ) =\sigma^2_b $$

따라서 이차원 확률 벡터 $y = (y_{ij},y_{ik})^t$의 공분산 행렬은 다음과 같다.

\begin{equation*} V_2(y) = \begin{bmatrix} \sigma^2_e + \sigma^2_b & \sigma^2_b \\ \sigma^2_b & \sigma^2_e + \sigma^2_b \end{bmatrix} = \sigma^2_e I + \sigma^2_b 1 1^t \end{equation*}

이제 위에서 논의한 공분산행렬을 일반적으로 $q$-차원으로 만드는 경우 함수에 공분산 행렬의 차원을 인자로 주고 지정된 차원의 공분산을 만드는 함수를 결과값으로 함수를 생각해 보자.

$$ V_q = \sigma_e^2 I + \sigma_b^2 1 1^t $$

이러한 공분산 행렬은 여러 차원의 공분산을 만드는 경우 각각 함수를 따로 만들지 않아도 된다. 아래의 R 함수 makeV 는 차원(q)를 인자로 받아 함수를 결과값으로 주는데 결과함수의 인자는 두 개의 분산으로 이루어진 벡터 $(\sigma_e^2,\sigma_b^2)^t$이다

3 Julia의 type 시스템과 method¶

위에서 볼 수 있듯이 Julia는 R과 달리 함수의 인자에 대한 type을 정확하게 지정하는 것을 기본으로 한다

function(para::Vector{Float64}) para[1]*eye(q) + para[2]*ones(q,q) end

물론 변수의 type을 지장하지 않을 수도 있지만 Julia의 언어설계 목표와 속도, 효율을 고려하면 대부분의 경우 인자의 type 을 지정해주는 것이 좋다.

만약 함수 csv3의 인자로 정수벡터 [1,2] 를 사용하면 어떻게 될까?

In [16]:
par=[1, 2]
csv3[par]

MethodError: no method matching getindex(::##3#4{Int64}, ::Array{Int64,1})

Stacktrace:
 [1] include_string(::String, ::String) at ./loading.jl:522

이렇게 오류가 발생한다. 그 이유는 인자가 정수벡터일 때 함수 csv3가 정의되어 있지 않기 때문이다. 보통 R 에서는 컴파일할때 인자의 형식을 자동으로 바꾸어주기 때문에 (type promotion) 이러한 오류를 보기 힘들다.

Julia에서도 기본적으로 type promotion이 이루어지지만 안전하고 효율적인 프로그래밍을 위하여 type을 지정하는것이 좋다.

다음과 같은 인자의 type이 정의되지 않은 함수는 type promotion이 자동으로 이루어 진다.

type을 엄격하게 정의하고 지정해주는 방식은 안전하고 효율적인 프로그램을 구현하는데 중요하다.

위에서 정의된 공분산 행렬을 계산하는 함수에서 분산이 음수가 되거나 공분산 행렬의 조건에 벗어난 결과가 나올 수 있다.

예를 들어보자. 실수로 분산의 값을 음수로 넣은 경우 결과가 공분산 행렬이 아닌 형태로 주어지게 된다.

Julia는 위와 같이 type의 정의를 parametrc type 등 다양한 방법을 통하여 구현할 수 있다.

function(para::T) where T

R이 느린 이유 중의 하나가 변수의 형식을 지정하지 않으면 내부적으로 여러 가지 작업을 거쳐서 그 형식을 맞추어 준다(conversion and promotion). 이러한 일련의 복잡한 절차 때문에 많은 경우 최적의 효율을 구현하지 못하는 것이다. 이러한 연유로 Julia를 사용하는 경우 변수의 형식(type) 지정을 잘 해주는 것이 속도, 효울, 안정성을 보장한다.

이제 compound symmetric 공분산 행렬의 composite type을 정의하고 안전한 자료의 생성을 다음과 같이 구현할 수 있다.

여기서 유의할 점은 $q$-차원 compound symmetric 공분산 행렬을 저장할 때 꼭 $q \times q$ Array를 이용하여 저장할 필요가 없다. 왜나하면 위에서 고려한 공분산행렬은 특별한 형태를 지니기 때문에

$$ V_q = \sigma_e^2 I + \sigma_b^2 1 1^t $$

차원 $q$과 두개의 분산성분만 저장하는 새로운 type을 지정하는 것이 편리하다. 예를 들어 100 차원의 공분산($q=100$)은 보통의 Array 로 저장하면 10000의 저장공간이 필요하지만 특별한 형태를 지니기 때문에 그에 대한 type을 지정해 놓으면 저장공간을 낭비할 필요도 없고 오류도 피할 수 있다.

아래는 struct를 통하여 새로운 composite type compsymcovmat를 지정하였다.

In [25]:
struct compsymcovmat
    n:: Int64       # dimension
    v:: Float64     # \sigma_e^2
    cov:: Float64   # \sigma_b^2
    function compsymcovmat(n:: Int64, v:: Float64, cov:: Float64)

4. 최적화 방법¶

4.1 비선형 함수 최적화¶

이제 정규분포에 대한 최대가능도 추정법(MLE)을 고려해 보자. 정규분포의 MLE는 아주 쉽고 또한 식으로 구할 수 있지만 여기서는 비선형함수의 최적화 함수를 사용하여 직접 구하려고 한다. 어떤 경우에는 비선형함수의 부분미분값, hessian을 구하여 더 효율적으로 해결하는 경우도 있지만 여기서는 문제의 확장과 해법의 일관성을 고려하여 이러한 정보를 이용하지 않을 것이다. 확률변수 $y_i$가 서로 독립이고 정규분포 $N(\mu, \sigma^2)$를 따른다면

$$ y= [y_1, y_2, \dots y_n]^t \sim N(\mu,\sigma^2 I_n)$$

최소화해야 하는 목적함수는 $-2 \log L$, 즉 로그가능도함수에 -2를 곱한것이다.

$$ -2l(\mu,\sigma^2; y) =-2\log L = \log [ det(2\pi \sigma^2 I_n)] + \frac{1}{\sigma^2}(y- \mu)^t\ (y- \mu) $$

위의 로그가능도함수는 자료 $y$를 포함하고 있고 평균 $\mu$와 분산 $\sigma^2$의 함수이다. 따라서 로그가능도함수는 자료 $y$를 함수 내에 가지고 있어야 한다.

NLOPT 패키지의 bobyqa(Bound Optimization by Quadratic Approximation)를 이용하여 MLE를 구해보자. bobyqa 방법은 부분미분함수를 사용하지 않는 방법이고 목적함수가 이차식(quadratic function)과 매우 다른 경우는 최적화가 어려울 수 있다.

먼저 R코드로 된 예제를 보자.

4.2 QR 분해를 이용한 최소제곱법 (Least square method)¶

회귀분석에서 사용되는 추정법인 최소제곱법은 그 해를 구하는 방법이 매우 다양하다.

$$ E(y|X) = \mu = X \beta \quad V(y|X) = V $$ $$ \hat \beta = \arg \min_\beta (y-X \beta)^t V^{-1} (y-X\beta) $$

최소제곱법은 만약 $y$가 정규분포 $N(X\beta, V)$를 따르고 $V$를 아는 경우 $\beta$의 최대가능도 추정법과 일치한다.

$V(y|X) = \sigma^2 I$ 인 경우에 대한 방법을 OLS(Ordinary Least Square)라고 한다.

$$ \hat \beta = \arg \min_\beta (y-X \beta)^t (y-X\beta) = \arg \min_\beta \| y-X\beta\|^2 $$

이러한 경우 OLS 추정량 $\hat \beta$는 다음 방정식의 근이다.

$$ X^t X \beta = X^t y$$

만약에 $n \times p (n>p)$ 행렬 $X$가 완전계수(full rank)이면(즉, $rank(X)=p$) $X^t X$의 역행렬이 존재하고 OLS 추정량은 다음과 같이 주어진다.

$$ \hat \beta = (X^t X)^{-1} X^t y $$

하지만 많은 통계계산 프로그램에서 실제로 역행렬을 계산하는 경우는 드물다. 실제 OLS를 푸는 알고리즘으로 많은 경우 QR 분해법을 사용한다. 행렬 $X$는 다음과 같이 분해할 수 있다.

$$ X = Q \begin{bmatrix} R \\ 0 \end{bmatrix} = \begin{bmatrix} Q_1 & Q_2 \end{bmatrix} \begin{bmatrix} R \\ 0 \end{bmatrix} = Q_1 R $$

여기서 $R$는 $p \times p $ 상삼각행렬(upper diagonal matrix)이고 $Q_1$과 $Q_2$는 각각 $n \times n$ 직교행렬 $Q$의 앞의 $p$ 열 과 뒤의 $n-p$열을 모아 놓은 행렬이다.

이제 $Q^t Q =I$를 이용하여 오차제곱합 $\| y-X\beta\|^2$를 다음과 같이 분해할 수 있다.

\begin{align*} \| y-X\beta\|^2 & = (y-X \beta)^t (y-X\beta) \\ & =(y-X \beta)^t QQ^t (y-X\beta) \\ & = \| Q^t y- Q^t X\beta\|^2 \\ & = \left \| c - \begin{bmatrix} R \\ 0 \end{bmatrix} \beta \right \|^2 \\ & = \| c_1 - R \beta \|^2 + \| c_2 \|^2 \end{align*} 여기서

$$ c=Q^ty=\begin{bmatrix} Q_1^t \\ Q_2^t \end{bmatrix} y = \begin{bmatrix} c_1 \\ c_2 \end{bmatrix} $$

만약 $X$가 완전계수 행렬이면 상삼각행렬인 $R$도 완전계수이므로 방정식 $ R \beta = c_1 $는 유일한 해는 다음과 같이 구할 수 있다.

$$\hat \beta_{OLS} =R^{-1} c_1$$

위의 방정식은 실제 역행렬을 구하지 않고 상삼각행렬의 성질을 이용하여 쉽게 구할 수 있다.

최소화한 후에 잔차제곱함을 분해하면 $$ \min_\beta \| y-X\beta\|^2 = \min_\beta \| c_1 - R \beta \|^2 + \| c_2 \|^2 = \| c_2 \|^2 $$

위의 결과를 통하여 잔차제곱합(SSE)은 $\| c_2 \|^2$을 알 수 있다.

$$ SSE = \| y-X \hat \beta_{OLS} \|^2 = \| c_2 \|^2 $$

위의 알고리즘을 R 과 Julia 프로그램의 예제를 통헤서 구현해 보자. 예제에서는 독립변수의 개수가 10개, 자료의 개수가 100개이고 독립인 오차는 표준정규분포 $N(0,1)$을 따르는 경우를 고려할 것이다.

function makeML(y::Vector, X::Matrix)
    return function(para::Vector, grad::Vector)
        p =length(para::Vector)
        β = para[1:(p-1)]
        σ = exp(para[p])     # sigma를 log scale 로 추정
        μ = X*β
        n =length(y)

        logl=0.0
        for i = 1:n
            logl += -logpdf(Normal(μ[i],σ),y[i])
        end
        return logl   
    end
end
