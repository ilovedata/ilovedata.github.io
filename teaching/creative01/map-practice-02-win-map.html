안내

공간자료인 지도(map)의 정보가 R에서 어떻게 구성되어 있는지 간단한 예제를 통해서 알아본다. 공간 및 지도자료의 구성과 분석에 대한 자세한 안내는 Spatial Data Science with R에서 찾을 수 있다.

필요한 라이브러리와 예제 서울지도 불러오기

먼저 필요한 라이브러리(library)들을 실행시킨다. 만약에 필요한 라이브러리가 설치가 되어있지 않다면 install.packages() 명령을 통하여 먼저 실행한다. install.packages()는 최초 한번만 실행하면 라이브러리가 설치된다. 그 이후에 사용할 때마다 library() 명령을 통하여 불러오면 된다.

공간자료의 구성

벡터데이터

공간에 대한 위치정보는 벡터(vector)형식으로 저장된다. 지도에서 한 곳의 위치를 나타낼 때 2개의 좌표, 즉 위도(latitude)와 경도(longitude)로 구성된 지리 좌표계(Geographic coordinate system) 를 사용한다.

그림: 위도와 경도

예를 들면 서울 광화문의 위치를 지리 좌표계로 나타내면 위도와 경도는 37.5759 N, 126.9768 E 이다. 서울시립대의 위치는 37.5839 N, 127.0589 E 이며 고속터미날은 37.5048 N, 127.0049 E이다.

지도의 위치를 나타내는 방식은 벡터 자료를 이용하며 위치는 점(point), 점을 연결하는 선(line)과 다각형(polygon) 형식으로 저장된다. 패키지 sp에서 정의된 공각 자료 형식은 다음과 같다.

*SpatialPoints

*SpatialLines

*SpatialPolygons

그림: 공간을 나타내는 벡터 자료의 형식

SpatialPoints

지도의 한 위치를 나타내려면 위에서 본 것과 같이 위도와 경도로 구성된 좌표자료가 필요하며 이를 R에서 SpatialPoints 형식으로 저장한다. SpatialPoints 형식은 패키지 sp에서 정의한 형식이다.

이제 광화문, 서울시립대, 고속터미널의 위치를 지도 좌표계로 나타낼 수 있는 SpatialPoints 형식으로 만들어 보자. 위도와 경도들로 구성된 벡터를 만들고 두개의 벡터를 cbind 함수를 이용하여 합친다. 함수 cbind는 두 개의 벡터를 열 방향으로 합쳐주는 함수이다.

longitude <- c(126.9768,127.0589, 127.0049)  # 경도
latitude <- c(37.5759, 37.5839, 37.5048)    # 위도
lonlat <- cbind(longitude, latitude) # 위도와 경도를 합친다.
lonlat
##      longitude latitude
## [1,]  126.9768  37.5759
## [2,]  127.0589  37.5839
## [3,]  127.0049  37.5048

이제 위도와 경도로 된 자료를 SpatialPoints 형식으로 만들어 보자.

ptr <- SpatialPoints(lonlat)
ptr
## class       : SpatialPoints 
## features    : 3 
## extent      : 126.9768, 127.0589, 37.5048, 37.5839  (xmin, xmax, ymin, ymax)
## coord. ref. : NA

객체 ptr는 SpatialPoints 형식(class)을 가진 자료이다. 이제 ptr에 들어 있는 좌표를 함수 coordinate 로 볼 수 있다.

coordinates(ptr)
##      longitude latitude
## [1,]  126.9768  37.5759
## [2,]  127.0589  37.5839
## [3,]  127.0049  37.5048

행해이제 객체 ptr는 가지고 좌표 자료를를 지구 상의 위도와 경도로 지정해야 지도상의 위치를 지정할 수 있다. 다음과 같이 함수 CRS 를 이용하여 좌표계의 투영법(projection)를 지정한다. 현재 세계에서 사용되고 있는 투영법는 하나가 아니라 여러 개이며 표준화되어 있지 않다. 우리는 이 과목에서 WGS84(epsg:4326)라는 가장 많이 사용되는 죄표계를 이용할 것이다 (Coordinate Reference Systems, 지도 투영법 참조))

proj4string(ptr) = CRS("+init=epsg:4326")
ptr
## class       : SpatialPoints 
## features    : 3 
## extent      : 126.9768, 127.0589, 37.5048, 37.5839  (xmin, xmax, ymin, ymax)
## coord. ref. : +init=epsg:4326 +proj=longlat +datum=WGS84 +no_defs +ellps=WGS84 +towgs84=0,0,0

이제 서울의 지도를 그리고 광화문, 시립대, 터미널의 위치를 빨간점으로 표시해 보자.

tm_shape(seoul0) +    # SpatialPolygonsDataFrame(서울지도)를 지정한다.      
tm_polygons(col = NA) +   # 서울지도를 그린다.
tm_shape(ptr)  +    # SpatialPoints(광화문,시립대,터미널)를 지정한다.
tm_dots(col="red",size = 0.5, alpha = 0.5) # 위치를 그린다.
## Linking to GEOS 3.6.1, GDAL 2.2.3, PROJ 4.9.3

SpatialLines and SpatialPolygons

점들을 연결한 공간 자료의 형식이 SpatialLines이며 선들을 연결하여 닫힌 도형을 구성하는 것이 SpatialPolygons이다.

이제 raster 패키지에서 제공하는 spPolygons를 사용하여 광화문, 시립대, 터미널의 위치를 이어서 공간다각형(SpatialPolygons)을 만들어 보자. 세점을 이어주는 삼각형이 만들어 진다. 선택 명령문 crs=CRS("+init=epsg:4326")는 투영법을 지정하는 명령이다.

plg <- spPolygons(lonlat, crs=CRS("+init=epsg:4326"))
## Warning in (function (coords, hole = as.logical(NA)) : less than 4
## coordinates in polygon
plg
## class       : SpatialPolygons 
## features    : 1 
## extent      : 126.9768, 127.0589, 37.5048, 37.5839  (xmin, xmax, ymin, ymax)
## coord. ref. : +init=epsg:4326 +proj=longlat +datum=WGS84 +no_defs +ellps=WGS84 +towgs84=0,0,0
tm_shape(seoul0) +    # SpatialPolygonsDataFrame(서울지도)를 지정한다.      
tm_polygons(col = NA) +   # 서울지도를 그린다.
tm_shape(plg)  +    # SpatialPolygons  plg를 지정한다.
tm_borders(col = "red") # 다각형를 그린다.

지도 객체(map object) seoul0은 어떻게 구성되어 있는지 자료의 구성과 구조를 각각 class 과 slotNames 함수로 파악할 수 있다.

지도 객체(map object) seoul0은 SpatialPolygonsDataFrame이라는 클래스(class)로 정의되어 있으며 이는 R에서 지도 자료를 저장하는 기본적인 자료 형식이다.

class(seoul0)
## [1] "SpatialPolygonsDataFrame"
## attr(,"package")
## [1] "sp"

또한 지도객체 seoul0 는 5개의 내부자료(slot 이라고 부른다)로 구성되어 있다. 이 내부자료는 함수 slotNames를 이용해서 어떤 이름으로 저장되어 있는지 알 수 있다. 내부자료 5개 중에서 data는 각 행정구역의 특성을 포함하는 데이터프레임(dataframe)이며 나머지 4개의 내부자료(polygon, plotOrder, bbox, proj4string)는 지도의 위치정보(점, 선, 경계 등)와 투영법(projection)에 대한 자료이다.

slotNames(seoul0)
## [1] "data"        "polygons"    "plotOrder"   "bbox"        "proj4string"

아제 서울시 구별 경계 지도를 그려보면

tm_shape(seoul0) +    # SpatialPolygonsDataFrame(서울지도)를 지정한다.      
tm_polygons(col = NA) +    # 서울지도를 그린다.
tm_text(text = "name",size=0.7)  # 구이름을 표시한다.

아래에서 사용한 함수 qtm은 아래와 같은 명령문을 간단하게 축약한 명령어이다.

함수 qtm를 이용하여 서울의 구경계지도를 그려보자. 지도를 그리는 경우 함수 qtm의 선택명령어 text = "name" 위에서 생성한 각 구의 한글이름을 출력한다. 또한 구 이름의 문자 크기는 선택명령어 text.size = 0.7에 있는 비율값으로 조정할 수 있다.

qtm(seoul0, text = "name", text.size = 0.7)

지도 그리기

2017년 구별 가구수를 함수 qtm을 이용하여 그림으로 그려보자. 가구수의 규모는 색깔로 표시되며 (선택명령어 fill ="pop") 범례(label)는 자동으로 출력된다. 선택명령어 title은 그림의 제목을 지정한다.

qtm (seoul0,  fill ="house", text = "name", title="2017년 서울시 구별 가구수", text.size = 0.7)
